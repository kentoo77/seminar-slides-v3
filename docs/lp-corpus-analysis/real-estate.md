| Google Maps API | 15+ | 21% | モデルハウス / 店舗一覧で頻出 |

**洞察**
- 「swiper + jQuery + WP + Zen Kaku Gothic New」が **注文住宅/工務店デフォルトスタック**。
- モダン化は **Next/Astro 陣営（SANU, TOMORE, rebita, architects.notahotel）** に集中。ブランド力のある企業ほど Headless 化が進む。
- **AOS 0%** は実務上の大きな知見：data-aos を混ぜた LP は「2020年頃」のシグネチャ。2026年の採るべきは「GSAP ScrollTrigger か、素の IntersectionObserver」。
- **Playwright / Puppeteer 取得の HTML では `cowcamo.jp/urucamo` が中身ゼロ**（完全 CSR）。Next.js/Astro 化した案件は HTML 解析では情報量が減る、というコーパス特性がある。

### 2.2 CTA / コンバージョン要素

| 項目 | 件数 | 比率 |
|---|---:|---:|
| 電話番号 / `tel:` / 0120 | 25 | 35% |
| `<form>` / HubSpot / Formrun 等 | 11 | 15% |
| LINE公式友達追加 | 7 | 10% |
| Instagram 連携 | 37 | 51% |
| 「資料請求 / カタログ請求」 | 14 | 19% |
| 「来場予約 / 見学予約 / モデルハウス / 完成見学会」 | 9 | 12% |
| 「無料相談 / お問い合わせ」 | 43 | 60% |
| 「完成見学会 / 内覧会 / オープンハウス」 | 7 | 10% |
| **ダブルCTA（資料請求 + 来場系）** | 10 | 14% |

**所見**
- 「お問い合わせ」（60%）は全ジャンル共通の最大公約数。だが **真の不動産LPでは「資料請求」「来場予約」のダブル**が正攻法（14%）。さらに「完成見学会」を独立CTAにするパターン（CHIME白州、松栄の家、neie）は工務店の鉄板。
- **Instagram連携 51%** は全ジャンル中でも高い。施工事例の視覚資産が Instagram と自然に結びつくため。
- **LINE公式 10%** は意外と低い。beauty-clinic (>60%) と比べると桁違い。不動産は LTV が高く1対1コンバージョン単価が高いので、SMS/電話/フォームの方がROI高い。
- 「無料査定」「買取」CTAは 7件（resale_brokerage サブタイプ）で必ず使われ、**QUOカード2000円分** プレゼント（フジ住宅）など景品訴求と組み合わせる。

### 2.3 コンプライアンス表記

| 項目 | 件数 | 比率 |
|---|---:|---:|
| 宅地建物取引業 表記 | 2 | 3% |
| 景品表示法 明記 | 0 | 0% |
| フッターコピーライト | 70+ | 97% |

- **宅建業免許番号の表記はほぼ見当たらない**（72件中検出できたのは2件）。これは「ブランドLP / コンセプトサイト」が主で、会社概要ページに免許を置き、LP本体からは省く設計。**法令違反ではないが、下位ページへの導線は必須**。
- 景品表示法の「消費者庁」ないし「不当景品類及び不当表示防止法」明記も検出されず。チラシ系LP（ジール住宅）でも明示なし。**不動産表示規約（公正取引協議会）の「徒歩1分=80m」明記**も見当たらないが、「徒歩◯分」表記自体は22件で使用。

### 2.4 物件カードの必須要素

実際に「物件カード」を持つ 13 件を分析すると、以下6要素が**ほぼ例外なく揃う**:

| 要素 | 出現率 |
|---|---:|
| 1. サムネイル画像（外観 or パース） | 100% |
| 2. 物件名 / プロジェクト名 | 100% |
| 3. 最寄駅 + 徒歩分数 | 92% |
| 4. 価格 / 価格帯（「◯◯万円」or「未定」） | 77% |
| 5. 間取り / 専有面積 / 土地面積 | 85% |
| 6. タグ / ラベル（「新築戸建」「リノベ済」「商談中」「SOON」等） | 92% |

**KANJU（関西住宅販売）の例（`6648_kanju-minoh.jp`）**
```
<h4 class="property-list__item__ttl">スマイルタウン箕面船場阪大前駅Ⅱ</h4>
<p>北大阪急行南北線「箕面船場阪大前」駅より徒歩５分 …</p>

```
→ swiper-slide にラップされ、横スクロール。**第一期◯区画 / 土地面積 / 徒歩分数**が3点セット。

**エスケイ住宅販売（`5484_donbraco.jp`）の例**
```

```
→ **価格の数字だけフォントを2サイズ大きく、単位「万円」は小さく**。これは ec-shop.md でいう「価格タグ」のパターンに近い。

- **セクション**: NEWS → **施工事例ギャラリー（横スワイプ）** → 家づくりの流れ → 暮らしを支える性能 → 家づくり相談会 → 事例名＝ **「悠の棲家」「翠蔭の住まい」** ←**詩的命名の代表例**。

- **学び**: 施工事例1つ1つに**俳号のような邸宅名を付ける**ことで、ブランドの世界観と検索流入（「悠の棲家 neie」）を両立している。これが「建築系」LPの典型語法。

### 5.3 「詩的命名」の語彙パターン

分析で確認できた事例名:
- neie: **「悠の棲家」「翠蔭の住まい」** ← 漢字1字+「の」+住まい
- BESS: **「栖ログ」「梺ぐらし」「ワンダーデバイス」** ← 造語 + ひらがな
- モリリフォーム: **「栞」** ← 1文字ブランド
- フジ住宅: **「ASHITA no KATACHI」** ← ローマ字＋コンセプト語
- CHIME白州: **「CHIME白州」** ← 音＋地名
- 松栄の家: **「松栄の家」「古民家放送局」** ← コーポ名＋の家
- TOMORE: **「TOMORE = ともあれ」** ← 古語＋現代ひらがな
- TNER: **「トナー（転貸）」** ← 機能語のカタカナ化

**語彙ルール**
1. **漢字1〜2字 + 「の」 + 家/棲家/住まい**（〜の家, 〜の棲家）
2. **擬音/造語**（栖, ワンダー, CHIME）
3. **古語/和語のローマ字化**（ASHITA no KATACHI, TOMORE）
4. **機能を抽象化した造語**（TNER=転貸, REVZO=Revolution Zone?）

→ **AI生成のネーミング候補**として、上記4パターンを使えば工務店の新LPに即流用可能。

### 5.4 ダブルCTA パターン

14%に見られる「資料請求 + 来場予約」の**ダブルCTA**は、**ファネルの段階別CTA**として機能:

| 段階 | 資料請求 | 来場/見学 |
|---|---|---|
| 認知 | 郵送カタログで興味喚起 | — |
| 検討 | カタログ2冊目で比較促進 | モデルハウス予約 |
| 意思決定 | (省略) | 完成見学会 / 現地訪問 |

**設計原則**: **固定CTAボタン**（画面右下 or ヘッダー）は**資料請求**にし、**セクション末尾CTA**は**見学予約**にする。フジ住宅(ASHITA no KATACHI)は `_title`→資料請求→相談会→カタログ請求の**全セクション末尾CTA反復**。

### 5.5 宅建業法 / 景表法 の表記の薄さ

- **宅建業免許番号の LP 内表記は 3%**。ほとんどは**フッターの会社情報リンク**にのみ存在。
- 「○○地方整備局長(1)第○○○号」等の免許番号は会社概要ページ送り。
- 景表法の「公正取引協議会の規約に則る」明記はゼロ。ただし「徒歩1分=80m換算」は大手（三井・一条・野村）で暗黙に守られている。
- **リスク**: ブランドLPとして「ふわっと」させる設計思想は分かるが、**1つでも価格/平米/駅徒歩を書くなら、公正競争規約の適用を受ける**。フリーランスLP制作の現場では「元ネタPDFの注記を全部コピペ→下部に小さく置く」のが安全策。

### 5.6 フォント使い分けの実態

- **不動産系**: 白 + 紺 (#1A2B4A 系) or 赤 (#C0272D)。情報の可読性優先。
- **リノベ系**: 白 + モスグリーン / テラコッタ。ナチュラル系。
- **賃貸/レジ系**: 黒 + ゴールド or ブロンズ。高級感。
- **公共/移住系**: 青 + 黄 + 白。親しみやすさ優先。

## 6. 新発見（ラウンド1で気づかなかったこと）

### 6.1 「体験型LP」が第3軸として台頭

前回は「建築系 vs 不動産系」で切ったが、本ラウンドでは **マドリル（診断ツール）、SANU（サブスク会員）、CHIME白州（移住体験ツアー）** という**体験をCTAに据える**LPが目立つ。これは数の上では少数（全体の10%程度）だが、**デジタルネイティブ世代の不動産LP**がここに集まっている。**従来の「物件情報を広告するLP」から「暮らし方を提案するLP」への移行**が始まっている。

### 6.2 NOT A HOTEL / SANU は HTML 解析不可能

完全 CSR の Astro / Next.js ベースでは、静的 HTMLの grep では中身が取れない。これは corpus 設計上の**死角**。今後は Playwright で render 後のHTMLを取得する補助スクリプトが必要。

### 6.3 「QUOカード」景品訴求は売却LPで必勝

`fuji-satei.com` では「査定依頼で **QUOカード2000円分**」「先着50名 **QUOカード500円**」の二段景品を FV 内に配置。**売主の重い腰を金銭インセンティブで引き剥がす**手法は2026年も現役。景表法的には「総付景品=取引価額の20%まで」のルールに抵触しないよう金額設計されているとみられる（査定=取引価額未確定だが、成約時価額の20%以下なので安全圏）。

### 6.4 「物件名ではなくエリア名で SEO する」ポータル設計

KURASHITO 仙台は**全物件カードに「仙台駅」「北仙台駅」**を明示。これはポータル系の SEO 王道で、「仙台駅 賃貸 デザイナーズ」等の**エリア+属性キーワード**を物件数だけ量産している。逆に注文住宅系は「◯◯市 注文住宅」よりも**ブランド名検索**を狙うため、エリア連呼はしない。

### 6.5 「コモンスペース」という語彙

CHIME白州で「コモンスペース」「共用畑」「店舗棟」が**分譲住宅の USP として前面化**。これは Cohousing (共同住宅) の概念が日本の分譲市場に入ってきたシグナル。今後の分譲LPでは「共用/共同/シェア」語彙が増えると予想。

### 6.6 Swiper は依然として不動産標準、だが使い方が進化

- **Swiper v11** (2024末リリース) の採用が観察される（dighome, kanju-minoh）。
- 使い方は「単純なカルーセル」から「**1カード 全画面 + Coverflow 風 3D**」へ進化。
- **Freemode + mousewheel** を組み合わせた「横スワイプ=縦スクロール代替」の実装も複数。

### 6.7 フォントの「Typekit 率」と「ブランド格」の相関

Typekit (Adobe Fonts) を使う 13件は、**ほぼブランド志向案件**（neie, architects, UDS, +SHIFT 等）。Google Fonts だけで済ませる工務店LPとの**分水嶺**がここにある。設計料金帯の分かれ目とも言える（高単価＝Typekit）。

## 7. design-mcp 使用方針（real-estate）

### 7.1 テンプレート設計の分岐

新しい不動産系LPを作る場合、以下の判定フローで生成モードを分岐:

```
質問1: LPのゴールは「物件を売る」 or 「ブランドを伝える」?
  → 「売る」 → 情報量型（サブタイプB/C/D）
  → 「伝える」 → 世界観型（サブタイプA/E/F）
  → 「体験させる」 → 体験型（G/H + マッチング系）

質問2: 物件数は1件 or 複数?
  → 1件 → プロジェクトLP（ex: CHIME白州, パークアクシス）
  → 複数 → ポータル型（ex: KURASHITO, KANJU）

質問3: ターゲットは BtoC / BtoB?
  → BtoC → 完成見学会 / 資料請求 ダブルCTA
  → BtoB (売主/オーナー) → 査定フォーム + 景品訴求
  → BtoB (法人/オフィス) → Case Study + Contact

質問4: 価格は明示? 非明示?
  → 明示 (チラシ系/ポータル) → 価格数字を大きく
  → 非明示 (ブランド系) → 「お問い合わせください」+ 価値訴求
```

### 7.2 生成テンプレート: 「中小工務店の注文住宅LP」

**推奨スタック**（最低コストで既存パターンに合わせる）:
- CMS: WordPress + 独自テーマ
- フォント: Noto Serif JP (FV) + Zen Kaku Gothic New (本文), Google Fonts
- JS: Swiper v11 + IntersectionObserver による fade-up
- 色: ベージュ #F5EFE6 + 墨 #1F1B17 + アクセント深緑 #2E4A3A

**セクション順序**:
```
1. Hero: 明朝タイトル1行 + 写真スライド3枚 + 「完成見学会予約」ボタン
2. 代表メッセージ (ていねい / 寄り添う / 暮らし)
3. 施工事例ギャラリー (Swiper 6カード + 「もっと見る」)
4. 性能 (UA値/C値/耐震/長期優良 の4数字を大アイコン)
5. 家づくりの流れ (ステップ7個)
6. お客様の声 (スライダー3件)
7. イベント情報 (完成見学会 / 構造見学会 / 相談会)
8. スタッフ紹介 (3〜5人)
9. お問い合わせフォーム + 資料請求
10. フッター (会社情報 / 宅建業免許 / SNS)
```

**詩的命名の自動生成ルール**: `[漢字1字: 悠/翠/栞/柊/楓/凛/結/詠/泉/栖] + の + [棲家/家/住まい]` から候補を5つ提示。

### 7.3 生成テンプレート: 「不動産ポータル / 物件一覧LP」

**推奨コンポーネント**: `property-card` を再利用可能コンポーネント化。
```html
<article class="property-card">
  <figure class="property-card__thumb">
    <img src="..." alt="物件名">
    <span class="property-card__tag">新築戸建</span>
  </figure>
  <div class="property-card__body">
    <h3 class="property-card__ttl">{{物件名}}</h3>
    <p class="property-card__access">{{路線}}「{{駅}}」駅 徒歩{{分}}分</p>
    <p class="property-card__price">{{間取り}} / <strong>{{価格}}</strong>万円</p>
    <p class="property-card__spec">専有面積 {{面積}}㎡</p>
    <a class="property-card__cta" href="...">詳細を見る</a>
  </div>
</article>
```

**禁則**: タグは**1カードに1つ**まで。色は 4色以内（新築/中古/リノベ済/商談中）。

### 7.4 生成時の禁則リスト（real-estate 特有）

- **景表法禁止語**: 「絶対安全」「完全防音」「業界最高」「日本一」（根拠明示なしはNG）
- **物件広告ルール**: 「駅徒歩◯分」は 80m/分で換算。「平米」は㎡表記。間取り表記は「3LDK」形式。
- **価格表記**: **「約」「〜」「から」 の曖昧表記は不動産表示規約で制限**。必ず「税込/非課税」明記。
- **写真**: ブランドイメージ写真と実際の物件写真を**明確に分ける**（「イメージ写真」のキャプションを入れる）。
- **施工事例の命名**: 個人情報（施主名、住所）を含めない。
- **宅建業免許**: フッターに必ず掲載。例: 「国土交通大臣 (◯) 第◯◯◯◯号」

### 7.6 design-mcp でやらないこと

- **物件情報を捏造しない**: ダミーテキストで「物件名」「価格」「駅」を生成する場合、必ず `[ダミー]` プレフィクスを付ける。
- **景品訴求の金額を勝手に決めない**: 「QUOカード2000円」等の**数字**はユーザー確認必須。
- **宅建業免許番号を生成しない**: 実在番号と衝突するリスクがあるため、フッターは**空欄またはプレースホルダ**とする。
- **写真は必ずプレースホルダ**: Unsplash や Pexels のAPIで CC0写真を引く、または「ここに物件写真」テキストボックス。

## 10. まとめ: 不動産LP の2026年スタンダード

### 10.1 3つの基本形

1. **建築系（世界観型）**: 明朝体 + 写真 + 詩的命名 + 完成見学会CTA
2. **不動産系（情報量型）**: 物件カード + 価格 + 駅徒歩 + 査定/問合せCTA
3. **体験系（第3軸）**: 動画 + 料金明示 + サブスク/診断/ツアーCTA

### 10.2 差別化のポイント

- **詩的命名**（〜の棲家, 〜の家, 〜の住まい）は建築系の必須装備。
- **物件カード6要素**（サムネ/名/駅/価格/間取/ラベル）はポータル系の鉄則。
- **ダブルCTA**（資料請求 + 来場予約）は 14% のみだが、CVR が最も高い設計。
- **QUOカード景品**は売却LPの秘密兵器。

### 10.3 テクノロジーの世代交代

- **AOS → ScrollTrigger → IntersectionObserver** への移行が進行中。
- **WP + jQuery** が主流だが、**Next/Astro** 陣営がブランド案件を独占。
- **3D/VR** は ROOV のような **iframe型 B2B SaaS** として外部化。LP本体には入らない。

### 10.4 design-mcp への実装優先度

| # | テンプレ | 優先度 | 理由 |
|---|---|---|---|
| 1 | **注文住宅/工務店（世界観型）** | 高 | 市場規模大、詩的命名パターンが再現しやすい |
| 2 | **不動産ポータル（物件カード型）** | 高 | 物件カード6要素がコンポーネント化しやすい |
| 3 | **賃貸ブランド（個別レジデンス）** | 中 | 共用部/専有部の表現が複雑、物件単位で別デザイン |
| 4 | **売却/査定LP** | 中 | フォーム UX と景品訴求が特殊 |
| 5 | **サブスク/体験型** | 中 | まだ事例少、テンプレ化リスクあり |
| 6 | **建築設計事務所（写真主義）** | 低 | 写真の良し悪しで決まるためテンプレ依存度が低い |
