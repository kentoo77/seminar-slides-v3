## 4. 共通パターン（ジャンル横断）

### 4-1. 「信頼」の構築装置（よく使われる順）

1. **No.1 バッジ + カッコ内で集計対象明示** （WealthNavi / FWD / GMO クリック / 第一生命）
2. **「調査期間・調査対象・出典」の脚注 `※1`** （景表法6条対応、モバイル保険 / 第一生命ステップジャンプ）
3. **創業年 / 実績年数** （`明治〇年創業`, `〇万件の相談実績`）
4. **認定・金融庁登録番号** （WealthNavi の第一種金融商品取引業者登録表記など）
5. **受賞バッジ**（価格.com 保険アワード、オリコン顧客満足度、J.D.Power）
6. **顔写真 + 署名**（税理士・弁護士・VCパートナー）
7. **Premium タグ（Gold/Platinum）の強調**（クレカ）

### 4-2. 景表法 / 金融庁ガイドライン対策の典型

- **No.1 の定義**: `（預かり資産・運用者数）` のようなカッコ内で集計対象を明記
- **「最短◯分」「◯日で」**: 条件付きとして `※審査状況により～` を必ず添える
- **無利息**: 「最大◯日間」で上限を明記
- **月々◯円**: `※40歳女性、非喫煙者優良体、払込方法クレカ月払` 等の**保険料試算前提条件**を脚注化
- **「シミュレーション」「診断」**: 結果に **「ご参考までにご案内した結果です。本契約の保険料とは異なる場合があります」** の免責
- **元本保証なし**（投資系）の明示
- **貸金業法 16 条 3 項**: カードローン・キャッシング LP で **「計画的なご利用をおすすめします」** 必須文言

### 4-3. タグ・計測の地層（重要）

多くの大手 LP は **3層タグ構造**:
1. **旧 GA**（`_gaq.push`, UA-）: 退任できないまま残存（セブン銀行、アイフル）
2. **GA4 + GTM v1**（本社管轄）
3. **GTM v2** + Adobe DTM（広告代理店管轄、キャンペーン単位）

この 3 層が並列し、1 LPに 3 本 GTM + Adobe DTM + Karte + Ptengine を同時配信することも珍しくない（東京海上あんしん生命、第一生命）。

### 4-4. CTA フローティングバーの構造

- 固定 bottom（SP）・サイド（PC）の2パターンがメジャー
- **3 つ並列**（申込 / 相談 / 資料請求）が生保の定番
- カードローンは**1 本集中**（申込 or 診断）
- 税理士・法律は**電話番号 + 相談予約**の 2 本

### 4-5. 「CTA → 別サイトへジャンプ」構造

金融 LP は **本物の申込フォームは巨大別システムで動作** するため、LP 内フォームは最小限。LP は「訴求 → ジャンプ」の役割に特化。
- 東京海上: `www2.tmn-anshin.co.jp/system/catalog/request01.asp`（ASP 古式）
- FWD: `reserve.fwdlife.co.jp`（予約SaaS）
- セブン銀行: `ib.sevenbank.co.jp/IB/...aspx`（IBM WebSphere 系）

## 5. 新発見（Round 1 との差分）

### 5-1. 「診断・シミュレーター = CVR 装置」仮説の修正

Round 1 では住友生命 Chakin の月5,000〜50,000円選択式を「最強 CVR 装置」と位置付けた。
Round 2 の94件全数分析で **「診断」言葉は 39% / 実装は 5%未満**と判明。

**修正版仮説**:
- **「診断ボタン」と「診断結果ページ」は分離する**のが業界標準
- LP 内に埋めるのはセブン銀行カードローンのような**"3〜5秒で終わる軽量診断"のみ**
- 本格的な保険料試算・ローン試算は**別システム（ASP）に外出し**して、計算精度責任を回避
- **LP 内シミュレーションは画像 or 表**で代替するケースが7割以上

### 5-2. 業種ごとのモダナイズ断層

- **生保** = 画像 + jQuery + 複数 GTM + Adobe DTM の**化石レイヤー**
- **ネオバンク / SaaS 採用** = Nuxt / Next / SPA + Lottie + GSAP ScrollTrigger の**最先端レイヤー**
- **採用LP（地銀/都銀）** = Nuxt + Noto Sans JP 400 の**テンプレ収束レイヤー**
- **税理士・法律・VC** = WordPress + 明朝 + 独自 CSS の**中堅士業レイヤー**

**同じ「金融」ジャンルの中に 4つの明確な技術地層**が並存。リニューアル周期は 5〜10 年単位で、全業種同時にモダン化は起きていない。

### 5-3. Consent Mode v2 の普及率は絶望的に低い

**94件中 明示実装は 2件**（careers.amt-law.com, pay.pocket-change.jp）。datasign CMP は検出ゼロ。OneTrust は Square / PayPay など**グローバル本社系のみ**。

→ 改正個情法（2024 施行）対応は**この業界では法律事務所の方が広告主より先行**という皮肉な構図。

### 5-4. VWO / A/B テスト：トップ1社のみ

**みんなの銀行（`3428`）だけが VWO + Adjust SmartBanner + Google Optimize + GA3 + GA4 の完全スタック**。ネオバンク単独の特殊ケースで、他 93件 には A/B テスト基盤が見当たらない。

→ **金融業界は A/B テスト文化が異常に薄い**。変更には法務・コンプラ承認が必要で、バリアントを出す運用コストが高すぎるため。

### 5-5. フォント多様性は意外に低い

Noto Sans JP 27件 / Montserrat 8 / Roboto 6 / Inter 5 / Poppins 4 / Lato 4 / Zen Kaku 3。**上位 7 フォントで 57件（61%）**をカバー。他ジャンル（ファッション、コスメ等）と比べて**和文・英文ともに選択肢が狭い**。理由は「読みやすさ > 個性」という業界要請。

### 5-6. 画像テキストの温存

全ジャンル横断でも最も**テキスト画像化率が高い**。FV から本文まで PNG/JPG で焼き込む事例は 20 件以上。A/B テスト・多言語化・SEO・アクセシビリティの全てに不利だが、「デザイン承認を画像で固めた方が社内が通りやすい」という組織論的理由で温存されている。

### 5-7. 発見できなかった項目

- **datasign Web-CMP** = 0件
- **Hotjar** = 0件
- **LINE タグ** = 0件（LINE 広告経由の計測は金融では許可が降りにくい？）

- FV: 事務所外観/パートナー写真 + 明朝体キャッチ
- Shippori Mincho / Noto Serif JP ベース
- 実績数字（創業年、相談件数、解決実績）

- CTA: 「無料相談」+ 電話番号 `tel:` の 2 点
- WordPress ベース想定

### 6-2. 絶対実装項目（Round 2 での必須リスト）

1. **No.1 表記の直後にカッコ内集計対象**（景表法）
2. **`※1` 脚注 + 下部の調査概要テーブル**
3. **3段CTA（商品LPの場合）**: 申込 / 相談 / 資料DL
4. **フローティング CTA は bottom 固定（SP）/ サイド固定（PC）**

8. **画像には必ず `alt`、見出しは `<h1>`..`<h3>` セマンティック**（生保の"画像焼き込み"を避ける）

### 6-3. 推奨しない事項（金融特有のアンチパターン）

- ❌ テキストを全て画像にする（古い生保パターン。アクセシビリティ / SEO / 翻訳に全て不利）
- ❌ jQuery 3.x に依存する（レガシー。Nuxt/Next/Astro 採用を推奨）
- ❌ GA3 (UA-) 併用（GA4 のみに統一）
- ❌ GTM を複数本並列する（承認フロー単位で分けたくなるが、1 本に集約する）
- ❌ FV に CTA がない（生保の画像のみ FV は CVR 無駄）
- ❌ 「診断」ボタンだけあって実装なし（別サイト遷移が 100%）← UX は悪い。LP 内に 5 秒診断を埋める方が CVR 高い
- ❌ Material Icons / Material Symbols の大量取り込み（重い）
