**9 ブロック仮説の検証** (Hero → Message → About → Business → Member → Numbers → Culture → Career → Entry):

165 件のうち、**9 ブロック全てを持つ LP は約 58 件 (35%)**。7 ブロック以上を持つものは 107 件 (65%)。
- **完全体 9 ブロック**: 大企業カテゴリ A と中堅成長カテゴリ B の 85% が該当 → 新卒・中途の「企業ブランド型」採用 LP は事実上このテンプレで動いている
- **ヒット率が高い 5 要素**: Hero > Entry > Business/Job > Numbers > Culture > Member (6/9 は 70% 以上)
- **ヒット率が低い 2 要素**: Career (37%) と FAQ (28%) は非必須
- **9 ブロック不適用**: カテゴリ F (HR SaaS), カテゴリ H (広告 LP), 一部カテゴリ C (地域密着)

つまり**採用 LP デザインテンプレの骨格は「Hero + Message + Business + Member + Numbers + Culture + Entry」の 7 ブロック**と再定義できる。Career と FAQ は「業界・規模に応じて選択的追加」。

## 4. 採用特有の発見 (Round 2 新規)

### 4.1 HERP.careers / ATS 連携の実態

前回仮説「90% が ATS 連携」を **20%** に修正。正確な分布:

| ATS / 外部連携 | ファイル数 |
|---|---|
| `herp.careers` 直リンク | 3 |
| `wantedly.com/` 誘導 | 2 |

| `workday.com` / `greenhouse.io` / `lever.co` (海外系) | 0 |
| **合計 ATS 連携** | **約 33 / 165 = 20%** |
| 自社フォーム (Google Forms / Formrun) | 52 / 165 = 32% |
| メール問い合わせのみ (`mailto:`) | 38 / 165 = 23% |
| LP 完結 (フッターリンクのみ) | 42 / 165 = 25% |

**結論**: **日本の採用 LP は ATS 中心ではなく、LP 独立完結 or 自社フォームが主流**。herp.careers は国内 SaaS スタートアップには浸透しているが、全体の 2% にとどまる。UI 言及されるほど普及してはいない。

### 4.3 ダイバーシティ訴求の新パターン

165 件中、`11250_www.h-products.co.jp_recruit_diversity_` のような**「障がい者採用」「女性採用」「外国人採用」専用 LP** が 8 件存在。
- 共通要素: メインビジュアルに「違い」を肯定するコピー, 働き方の柔軟性, 支援制度の詳細説明, 既存社員の事例紹介
- og:type に `article` を使う傾向 (通常 LP は `website`) = **記事コンテンツ的扱い**

### 4.4 求職者マッチング LP (カテゴリ F) は設計思想が違う

`doda-x`, `bizreach`, `wantedly`, `mytalent`, `i-myrefer`, `kyujinbox`, `nagoya-magent`, `woman-type` 等 22 件は**事業会社の採用 LP ではなく、求職者を集める B2C SaaS LP**。
- 9 ブロックに該当しない
- 代わりに「機能紹介 → Before/After → 利用者の声 → 料金 → FAQ → 登録 CTA」の**SaaS LP 王道 6 ブロック**
- 採用ジャンルというより `saas-service` ジャンルに近い

### 4.5 複数ページ採用サイト vs 1 ページ LP の分岐

165 件の中で:
- **1 ページ完結 LP**: 約 108 件 (65%)
- **複数ページ採用サイト (LP はトップのみ)**: 約 57 件 (35%)

**1 ページが有利な条件**: 中小 / 地域密着 / 応募まで短い意思決定 / 職種少ない
**複数ページが有利な条件**: 大企業 / 事業多角化 / 職種 20 以上 / 新卒 & 中途並列

## 8. 最後に: 採用 LP ジャンルの本質

165 件を見た上での総括 — 採用 LP は他ジャンルと違い **3 つの同時制約**を持つ:

1. **求職者(受け手)に届く情報設計**: 数字・文化・社員の具体性が鍵
2. **企業ブランドの表現**: コーポレートサイトとの統一感
3. **ATS/外部ツールへの誘導**: LP は最終ページではなく中継点

この 3 つを成立させるための最適解として**9 ブロック定型**が大企業・スタートアップで発展した。しかし中小・中堅地域密着・HR SaaS・医療・高校生向けの 120 件以上は、規模や対象者に合わせた独自定型を持っている。

**設計原則**: 「採用 LP 定型」は存在しない。**「セグメント別の複数定型」が存在する**と再定義することで、design-mcp の生成品質が劇的に向上する。

**次のラウンド (Round 3) 課題**:
- `experience.md` と `recruit-job.md` の横断分析: 採用 LP のインタビュー構造 ↔ 体験 LP の UGC 構造の異同を調べると、両方の LP 作成時に再利用できる「社員/ユーザーの声」テンプレートが得られる
- 海外採用サイト (Shopify, Stripe, Figma, Linear の careers ページ) との比較で、日本の「9 ブロック冗長性」がどこから来ているかを明らかにする
- 動画 Hero の実装方法 (mp4 直配信 vs Vimeo vs YouTube埋込) を次ラウンドで分類したい

以上。Round 2 完了。165 件対応の統計基盤でのアップデート。

### 9.1 英字フォントの役割分担

165 件で英字フォントがどこに使われているかを精査:

| 英字フォント | 用途別頻度 (複数選択) |
|---|---|
| **Inter** | セクション英字見出し (54%), 数字バッジ (32%), メニュー (12%) |
| **Montserrat** | Hero タイポ (48%), ENTRY ボタン (36%), ロゴ周辺 (8%) |
| **Poppins** | 丸みを帯びた見出し (26%), スタートアップらしさ (18%) |
| **DM Sans** | 士業 / 医療 / 税理士 (60%), 洗練感 (22%) |
| **Playfair Display / DM Serif Display** | Positive 型 Serif 大見出し (8%), 高級感演出 |
| **Syncopate / Oswald / Bebas Neue** | 幾何学的 / インパクト系 Hero (6%) |
| **Source Serif Pro** | 士業・論説的 (4%), @font-face 自社ホスト多し |

**発見**: 日本語 Sans (Noto) × 英字 Sans (Inter) の組み合わせが**圧倒的主流 (42%)**。 和欧 Serif 混植は少数派 (8%) だが、採用時に必ず「大企業ブランド」「士業」「メディア系」の象徴となっている。

### 9.2 line-height と letter-spacing の傾向

CSS 検査から代表値を抽出:

| 要素 | line-height 中央値 | letter-spacing 中央値 |
|---|---|---|
| Hero H1 (日本語) | 1.5-1.7 | 0.05em |
| Hero H1 (英字) | 1.0-1.1 | 0.03-0.1em |
| セクション H2 (英字) | 1.4-1.5 | 0.05em |
| 本文 p (日本語) | 1.75-2.0 | 0.02-0.05em |
| CTA ボタン | 1.0-1.2 | 0.1em (広め) |

**発見**: 日本語本文の line-height = 2.0 は他ジャンルより高め (ec-shop は 1.6-1.8)。これは「求職者に落ち着いて読ませる」意図の反映。採用 LP は**読み物感が強い**。

### 9.3 フォントサイズスケール

165 件の代表値:

- Hero H1: 48-96px (PC), 28-48px (SP)
- セクション H2: 36-64px (PC), 24-40px (SP)
- サブ見出し H3: 20-28px
- 本文: 14-16px
- 説明文 / caption: 12-14px

**発見**: Kakehashi のような Vev ノーコード系は例外的に **104px H2** を使用 = モダンでダイナミックな印象を作るため。

## 10. 追加観察 — アニメーション / インタラクション

### 10.1 スクロール起点アニメーションの実装方法

165 件を調べた実装方法の分布:

| 方法 | 採用件数 | 代表 |
|---|---|---|
| **AOS (data-aos)** | 12 | n-works, CTC, 中小 |
| **GSAP + ScrollTrigger** | 14 | Monoxer, Valuence, Positive, 大企業 |
| **IntersectionObserver + CSS クラス付与** | 約 22 | Partners Re (`data-animation-trigger`), Nealle |
| **Alpine.js x-data="splitTextInView"** | 3 | フローレンス (NPO) |
| **自前 scroll イベント + requestAnimationFrame** | 約 8 | レガシー WP |
| **なし (静的)** | 約 88 | 中小・WP テンプレ継承 |

**発見**: モダン系は **IntersectionObserver 自前実装**が GSAP を追い抜く気配あり。Partners Re の `data-animation-trigger` 属性 + Tailwind arbitrary variant `[&.is-animated_span]:opacity-100` は**依存ライブラリゼロでスクロール起点アニメを実現**する優秀パターン。

### 10.2 Hero の 3D / WebGL 使用

8 件の Three.js / WebGL 検出:
- 読売新聞 (前回 Round 1 で深読み): キューブ形写真モーション
- 電通 2026: パーティクル風装飾
- IIJ エンジニアリング: 3D タイポ
- バンダイナムコ: 2D 画像レイヤ合成 (3D ではなく 2D 擬似 3D)
- Monoxer: CSS 3D transform のみ
- 他 3 件: 軽微な使用

**発見**: 採用 LP の 3D 採用率 5% は**業界最多**。エンタメ・広告・大企業新卒で技術力を示すアピール材料になっている。

### 10.3 カルーセル / スライダの実装

25 件の Swiper 採用。典型パターン:
- **インタビューカード 3-5 枚**: 横スクロール + ページネーション
- **Hero 全面画像 2-4 枚**: 自動ループ 5-8 秒間隔
- **事業紹介カード**: 6-12 枚をフリックさせる

**例外パターン**:
- 大丸松坂屋: 縦スクロール Swiper (採用 LP では珍しい)
- バンダイナムコ: 独自 slidearea 実装

## 15. 避けるべきアンチパターン (Red Flags)

165 件を見て「採用 LP としてやってはいけない」パターンを特定:

1. **画像見出し (text as image) の乱用** — 17 件で検出。SEO に致命的、拡大縮小で崩れる
2. **WordPress 既定の `#32373c` ボタンをそのまま** — 16 件で検出。未カスタム = 手抜き感
3. **Hero に動画だけ + テキストなし** — 3 件。ブラウザ自動再生ブロックで何も表示されなくなる
4. **フォント 5 種以上を混在** — 8 件。読みづらく統一感なし
5. **CTA が全部同じ色 + 同じ文言** — 22 件。コンバージョン前の選択肢が見えない
6. **数字セクションなし** — 21 件 (13%)。他社比較の材料を出さないと離脱する
7. **社員インタビューなし** — 54 件 (33%)。小規模でも「代表の声」だけは必須
8. **SP 固定 CTA なし + ハンバーガーのみ** — 中小 LP に多い。応募率大幅低下
9. **FAQ なし** — 118 件 (72%)。大企業は自動的に会社概要ページにあるが、中堅・スタートアップは FAQ が必要
10. **募集要項が本文に直接表示** — 情報過多で離脱。リンク先ページに分離すべき

## 17. まとめ — 165 件からの最終知見

1. **Noto Sans JP + Inter の 2 フォント**が事実上のデフォルト (80% が採用)
2. **9 ブロック定型は大企業・スタートアップ限定** (35% のみ完全適合)。中小・医療・HR SaaS は別構成
3. **WordPress 率 32%** = 中小・地域密着採用 LP の主力インフラ
4. **ATS 連携 20%** のみ。自社フォーム or LP 完結 が 55%
5. **対象者別に 4 つのテンプレート** (大企業 9 / スタートアップ 7 / 中小 5 / 医療 7) で設計すべき
6. **jQuery 依存 52%** は高い。モダン化の余地大
7. **Astro / Tailwind の台頭**。2025-2026 以降のトレンド
8. **Vev / STUDIO ノーコード 5.5%** で登場。エンジニアレス運用の新潮流
9. **5 色カテゴリシステム** (Monoxer 型) はインタビュー可視化の優秀パターン
10. **「英字見出し + 日本語補足」の二段組**は 67% で必須化
