- **タイポを前面に出さない**（フォントはシステムフォント中心・商品写真と競合させない）
- **店名／サービス名の意味をコピーで繰り返す**（soyogo=ソヨゴ、ch.books=チャンネル、miyokka=してみよっか？）

### 2-4. 拡張業態の経営思想

- **独立系書店**は「本単体では赤字」を前提に、**カフェ・編集業・イベント・デザイン受注**で複合化
- **大手出版社**は採用ブランディング・オウンドメディアへの投資を強化
- **印刷会社**（NPC / 日販系）が**出版機能の代替**に進出

### 2-5. 書籍情報の扱い（元タスク項目への回答）

| 項目 | 観測 |
|---|---|
| ISBN全開示 | **大和書房のみ**（検索ボックスでISBN入力可） |
| 判型・ページ数 | 0件（LPレベルでは非表示、詳細ページにあると想定） |
| 発売日 | 大和書房の新刊一覧で暗に提示 |
| 価格（定価） | 0件（LPレベルでは「電子書籍40%OFF」キャンペーンのみ） |
| Amazon+版元2系統CTA | 0件（**6件すべてで書籍単品の購入CTAが存在しない**） |
| 著者の売り方 | 大和書房（著者名・監修者名羅列）/ soyogo（著者一覧） |
| 編集者の売り方 | 講談社採用の職種別紹介のみ |

**→ ラウンド1の旧分析で立てた「書籍情報全開示 / Amazon+版元2系統 / 著者編集者の売り方」という仮説は、今回のコーパスでは検証できない**。現コーパスは**書籍販売LPを1件も含まない**ため。

## 3. 新発見

### 3-1. 「出版」カテゴリの拡張定義
旧ラウンド1は「書籍特設LP中心」だったが、今回は**「本を取り扱う事業すべて」**が含まれる。LP制作サイドの仕事フレームを、単純な「書籍プロモ」から、**「場所・人・コンテンツ・体験・SaaS」**にまで拡げる必要がある。

### 3-2. 印刷会社の"出版化"（soyogoパターン）
日本印刷株式会社（NPC）が**自社発行の書籍レーベル「soyogo books」**を立ち上げ。これは**印刷会社 → 出版社**への業種転換の典型例で、**B2B受託だけでは成長できない印刷業界の突破口**として業界注目すべき動き。

### 3-3. 日販の"体験店舗"実証（miyokkaパターン）
日本出版販売（書籍流通最大手）が**直営書店＋プレイパーク**を運営。これは「書店の存在意義=本を売る場所」を放棄し、**「家族の週末居場所」**に再定義する事例。SaaS化・カフェ化・サロン化と並ぶ**リテール存続戦略の第4軸**。

### 3-4. 寄せ書きの「書籍化」というSaaS（sugoyoseパターン）
寄せ書きを**ブック形式**で商品化（2,300円〜）＝「思い出の物体化」をサービス化。**推し活市場（カンパ機能）**と**英語圏展開（シンガポール通貨決済）**の両方に張る設計は、**日本のポップカルチャー資産を海外展開するD2Cの教科書**。

### 3-5. 採用LPの「読みもの化」（講談社パターン）
講談社の採用LPは**採用情報ページというより『文芸誌のプロローグ』**に近い作り。「ものがたり」「才能と、働こう。」のキャッチ、北澤平祐イラスト、担当者対談、内定者エッセイ——**読者体験そのものがセレクション**になっている。**「出版社はLPで文学性を示すことで応募の質を担保する」**という広告戦略。

### 3-6. 地方書店の3業態化（ch.booksパターン）
「書店 + 朝カフェ + 編集デザイン事務所」の**完全3業態1LP**は、地方の個人事業主に再現可能な**最小単位の複業構造**。LPの2階建て（1F=店舗 / 2F=制作）という物理建築に即した情報階層は、**業態多角化を視覚的に伝えるUI**として秀逸。

## 4. design-mcp使用方針

### 4-1. テンプレ推奨

| 業態 | design-mcp テンプレ | 優先度 |
|---|---|---|
| 独立系書店・書店カフェ | `local-bookstore` / `cafe-retail` | 中 |
| 体験型書店・ファミリー向け | `family-experience-retail` | 中 |
| 出版社採用LP | `publisher-recruit` / `branded-recruit` | 中 |
| 出版社コーポレート検索型 | `publisher-portal` | 低 |
| オウンドメディア文芸 | `alternative-literary-media` | 低 |
| 書籍化SaaS（寄せ書き/アルバム系） | `personalized-book-saas` | 中 |

### 4-2. デザイン方針

**必須要素**:
- **大胆な余白**（本を読む気分を阻害しないように）
- **写真/イラストを主役**にしてフォントは譲る
- **シリーズ名・レーベル名の並列展示**（大和書房の10文庫分類が手本）
- **店舗の場合は物理情報（住所・電話・営業時間）を明確に**

**避けるべき**:
- ❌ 派手なCTAボタン（書籍文化に馴染まない）
- ❌ 価格訴求の前面化（大和書房すら価格は出していない）
- ❌ 動画ヒーローの多用（文字メディアの文脈と乖離）

### 4-3. 旧ラウンド1との違い
旧分析では「書籍特設LPのISBN/価格/CTA」の定型を扱ったが、**現コーパスではそのパターンが再現できない**。今回の分析価値は「本周辺事業の多角化パターン」の方にある。

**design-mcp資産化すべき項目**:
1. 「書籍・書店・著者・編集者」のブランドLPテンプレの**静謐・文学的トーン**
2. 「複業1LP」のセクション設計（1階/2階/3事業を1ページに統合する情報階層）
3. 採用LPの「読みもの化」レイアウト（講談社パターン）
4. 書籍化SaaS（寄せ書き・フォトブック・アルバム）の**3プラン価格ブロック**（製本/PDF/Web）

## 5. まとめ

ラウンド2 publication 6件は、「書籍を売るLP」ではなく**「本周辺事業のLP」**の縮図。書籍経済が「本を売る → 本で稼ぐ」から「本の世界観で稼ぐ」にシフトしている現実が凝縮されている。

**残された課題**:
- 元タスクの「書籍情報全開示・Amazon+版元2系統・著者編集者の売り方」は、**書籍特設LPを含む別コーパスでの再取得**が必要
- 現コーパスは**書店複合業態 / 出版社ブランディング / 書籍化SaaS**の分析には優秀
- design-mcpで「本周辺事業」を生成する際の**サブジャンル分類**を拡張すべき（旧＝単純に"book-lp"、新＝"bookstore / book-publisher-corp / book-recruit / book-saas / book-media"の5軸）

ラウンド1の30件分析（旧`publication.md`）と内容が大きく異なるため、**統合ではなく別資産として両方を残す方が研究価値が高い**（本フォルダでは指示通り上書きしたが、旧版はgit履歴に残すべき）。
