## 3. 共通パターン集（71件から抽出）

### 3-1. "○○を知る" 構文（産業LP標準）

71件中、以下のような「Know」系ナビを持つサイトは**21件**：
- `会社を知る / 仕事を知る / 人を知る / 地域を知る`（東組、ムネカタ、パナInd、ケミコン、サンドラッグ）
- `東組を知る / 働く人を知る / 働く環境を知る`（東組）
- `想いを知る / 技術を知る / 会社を知る / 職種を知る`（パナInd）
- `奥村組について / キーワードで知る奥村組`（奥村組）
- `日本ケミコンを知る / 日本ケミコンで働く`（ケミコン）

**この"Know"構文はR1で見逃されていた最大の共通項**。採用LPのナビ設計において**「○○を知る」は事実上の標準**になっている。理由は：
- 動詞が「知る」で統一される＝**受け手側（知らない候補者）の視点**
- 複数ナビを作ってもリズムが崩れない（全部に「を知る」が付く）
- SEOで「○○ 採用 知る」で入り口になる

### 3-2. 英日2段組見出し（15件以上で確認）

`ABOUT / 日本ケミコンを知る` 型の英語大見出し＋日本語小見出しの組合せが広く見られる。これは**「大手っぽく見せる装置」**として機能。実装は：
- 英字フォント（Inter/Public Sans/Roboto/Poppins等）で大見出し
- Noto Sans JP で日本語を細く小さく下段に置く
- **英字は装飾、日本語が本体**という関係

### 3-3. 「3知り」パターン

`会社 / 仕事 / 人` の3分類でナビを組む地方BtoBは極めて多い。
- 4396 東組：`東組 / 働く人 / 働く環境 / 地域`（4知り）
- 4599 ムネカタ：`会社 / 仕事 / 人 / 環境`（4知り）
- 4505 パナInd：`想い / 技術 / 会社 / 職種`（4知り・変奏）
- 4424 JA愛知：`仕事内容 / 採用情報 / 説明会`
- 4626 澤村：`会社 / 仕事 / 人 / 環境`
- 4442 鶴田電機：`会社 / 仕事 / 人`

「3知り or 4知り」は**地方BtoB採用LPのデフォルト構造**。R1では「10ブロック骨格」として提示したが、実際のナビゲーションレベルでは**3-4分類が標準**で、10ブロックはその内訳（下層ページ）と捉えるべき。

### 3-4. "数字で見る○○" セクション

33件（46.5%）が採用している。内容は定番で：
- 社員数 / 平均年齢
- 男女比
- 平均残業時間
- 有給取得率
- 新卒・中途比率
- 離職率 / 勤続年数平均
- 資格取得率
- 育休取得率（女性 / 男性）

**採用候補者の知りたい順**に並ぶ：健康・時間・金・成長の順。「売上・利益」は後ろの方か、そもそも出さない（採用候補者には関係ないから）。

### 3-5. プロジェクトストーリー構文

**「プロジェクト名 + 一文ドラマ」**を3-5個並べる構文。例：
- ムネカタ：`Go innovation 誰も見たことのない技術を、福島から。/ 半田ごてからの快進撃！進化する溶着システム`
- ロート：`ボラギノール越境ECプロジェクト / メラノCC ディープクリア酵素洗顔 開発プロジェクト`
- スター精密：`プロジェクトストーリー`セクション
- 奥村組：`プロジェクトストーリー`（動画付き）
- 三井物産：`プロジェクト紹介`（複数）

ドラマ化された1行タイトルは**記事タイトル風**に統一される（「技術の説明」ではなく「困難の克服」）。これは**NHK『プロジェクトX』の言語論**を踏襲している。

### 3-6. 採用CTAの3段階配置

71件で観測された採用LPの典型CTA配置：
1. **弱CTA（興味層）**: `カジュアル面談 / 資料DL / Company Deck / マイページ登録`
2. **中CTA（検討層）**: `説明会予約 / OB訪問 / インターンシップ申込`
3. **強CTA（志望層）**: `本エントリー / 募集要項 / 選考申込`

R1の「強CTA1個」説は否定され、**3段階のリードステップが標準**。これはSaaSマーケティングの"リード温度管理"を採用領域に持ち込んだ結果。
採用LPのCVR設計は**セールスLPより複雑**になっている。

### 3-7. "社員の顔写真 × 笑顔 × 現場"トリオ

71件中、Interviewセクションを持つ39件は全て**社員の顔写真＋笑顔＋現場（工場・建設現場・店舗）**の3要素を満たす。フォーマルスーツのポートレイトは皆無。重要なのは：
- **作業着や制服を着ている**（職場のリアル）
- **表情はナチュラル**（オフィシャルポートレイトではない）
- **背景に道具・製品・機械がある**（仕事の実物）

Canon/Sony/FujifilmのRGB校正済み写真ではなく、**ドキュメンタリー的なフォトグラフィ**が好まれる。プロ写真家と取材ライターのチームで制作されることが多い。

### 3-8. 創業年の3桁強調

71件中で創業年を前面に出しているサイト（深読み確認）：
- Bluestorm：**1935**年創業
- 奥村組：**1907**年創業（100年以上）
- 日本ケミコン：**1931**年創業
- スター精密：**1950**年創業
- ロート製薬：**1899**年創業（125年）
- 山﨑建設工業：**創業から百年**
- 三井物産：**1876**年創業
- パナソニックInd：パナソニック株式会社 **1918**年創業

"西暦4桁 + 創業" の組み合わせはインフラLPの**信頼の刻印**。数字は大きなフォントで、時には**左側に縦組み**で配置される。

### 3-9. YouTube動画のHero配置

71件中21件（29.6%）がYouTube埋め込みを使用。そのうち**14件はHeroに大きく配置**：
- 工場・建設現場の俯瞰ドローン映像
- 製品稼働シーン
- 社員の作業風景
- 3分自己紹介ムービー

**Hero動画は"動き"でいきなり信頼を届ける**。1秒で「この会社は動いている・本物である」と伝わる。写真だけのHeroより心理距離が縮まる。ただし再生自動・ミュート・ループが必須（音は出さない）。

### 3-10. "弱いブランド × 強い言い換え" のパターン

多くの産業LPは、業界的に地味な自社を**強いメタファー言語**で塗り替えている：
- スター精密 → "SMALL GIANTS"（世界を支える小さな巨人）
- Space BD → "宇宙商社®"
- 日本ケミコン → "CHEMI-CONNECT"（世界に触れる）
- ムネカタ → "Go innovation"（誰も見たことのない技術）
- 石坂グループ → "里山再生企業"（元は産廃業）
- MonoRevo → "ものレボ"（ものづくり革命）
- Tential → "Be Our Buddy"（仲間になろう）

これらは全て**「既存の業種名を避け、新しい言葉で自分を定義する」**作業。キャッチコピー制作ではなく**業態リネーム**のレベルで行われている。これが産業LPを読ませる最大の仕掛け。

### 4-2. R2で新たに発見した命題

1. **「人」の支配的地位**: 83.1%の出現率で、どのキーワードより高い。インフラLPは"人を前面に出す採用LP"である
2. **「○○を知る」構文**: 21件以上で観測。産業LP標準ナビの基盤
3. **「3知り / 4知り」構造**: 会社/仕事/人（/環境/地域）の4分類が標準。10ブロックの上位概念
4. **Next.js優勢**: 47.9%で jQuery(42.3%)を抜く。**モダンスタック**が産業LPに侵入
5. **業態リネーム手法**: 弱いブランドを新造語で再定義する（"SMALL GIANTS" "宇宙商社®" "CHEMI-CONNECT"）
6. **3段階リード漏斗**: カジュアル面談→説明会→エントリーの段階化が採用LPの主流
7. **"数字で見る○○"の定型化**: 46.5%。KPIは健康・時間・金・成長の順
8. **地方ゼネコンの英字ブランディング**: 山﨑建設工業のArvo、東組のZen Kaku、など地方中堅でも大手級のタイポ設計
9. **プロジェクトストーリーのドラマ化**: "プロジェクトX"言語論。技術そのものを語らず困難克服のナラティブに変換
10. **Hero動画の信頼機能**: 29.6%（21件）がYouTube埋め込み、うち14件がHero配置。1秒で"動いている会社"を証明

### 4-3. 業種別に細分化されたサブパターン

## 5. design-mcp 使用方針

### 5-1. テンプレート選択の意思決定

**ステップ1: 業種カテゴリ判定**

生成依頼が来たら以下を先に確認：
- ゼネコン / 建設 / 土木 → **A型**（3-4知り + 濃紺 + ドローン動画）
- 大手メーカー → **B型**（4知り + ブランドカラー + Cross Talk）
- 地方中堅 → **C型**（3知り + 濃紺/赤 + インタビュー動画）
- SaaS製品 → **D型**（Problem-Solution + 料金 + CTA）
- BtoCインフラ → **E型**（セールスLP準拠）
- ブランドストーリー → **F型**（歴史 + 哲学 + IR）
- 研究室 → **G型**（Publications + Members）
- 伝統工芸 → **H型**（プロダクト哲学）

**ステップ2: 目的判定**
- 採用（74.6%の既定値）→ A/B/C型
- 発注獲得 → 既存のS型（セールス）ではなく**D型 or E型**
- ブランディング → F型
- 研究者募集 → G型

**ステップ3: カラーパレット選択**

濃紺を使う場合：
```
#002F7C → プライマリ（深海）
#0B358A → セカンダリ
#4CA0F0 → アクセント（海面）
#FFFBFB → ベース
#E5EAF0 → グレー補助
```

赤を使う場合（ムネカタ型）：
```
#AD0306 → プライマリ
#1A1A1A → テキスト
#F5F5F5 → ベース
#E60012 → アクセント（ロート型）
```

ベージュを使う場合（自然系・weazer型）：
```
#E1E1D8 → ベース
#222222 → テキスト
#999999 → 補助
#D0D0D0 → ライン
```

### 5-2. フォント標準

**デフォルト選択**：
- 英字：**Inter**（19件で最多、最安全）
- 日本語：**Noto Sans JP** 400/500/700 + システムフォントフォールバック

**代替（重厚感が必要な場合）**：
- 英字：Montserrat（4件）or Public Sans（Space BD採用）
- 日本語：Zen Kaku Gothic New or Zen Kaku Gothic Antique

**避けるべき**：
- Playfair Display、Cormorant Garamond等のセリフ系（ブランドLP/ウェルネスLP向け）
- 明朝体全般（産業LPで2.8%のみ）
- Roboto Mono等の等幅（SaaS製品でもコードブロック以外は不要）

### 5-3. セクション必須/推奨リスト

**絶対必須（80%以上の採用率）**：
1. Hero（動画または工場写真）
2. Works / 実績一覧
3. Interview（社員紹介、顔写真＋現場）
4. Footer（住所、電話、拠点、許可番号）

**強く推奨（50-80%）**：
5. Recruit / Entry CTA（3段階リード漏斗）
6. Company / About（数字で見る○○）
7. News（更新の証明）

**業種依存（20-50%）**：
8. Message（社長メッセージ）← 大手採用で必須
9. Business / 事業紹介 ← グループ会社で必須
10. Strength / 強み ← 差別化が必要なB型で必須

**オプション（20%未満）**：
11. SDGs / サステナビリティ ← 大企業バッジ
12. Cross Talk / 座談会 ← 大手採用の差別化装備
13. Project Story ← ドラマ化された技術紹介
14. プラン / 料金（SaaS・BtoCインフラのみ）

### 5-4. コピーライティング指針

**見出しの変換ルール**：
- NG: "会社概要" "事業内容" "社員紹介"
- OK: "○○を知る" "○○で働く" "○○の仲間"
- 英日2段組（ABOUT / 私たちについて）で包装

**業態リネーム**：
- "旋盤メーカー" → "SMALL GIANTS"
- "部品会社" → "CHEMI-CONNECT（世界に触れる）"
- "産廃業" → "里山再生企業"

**キーワード優先度（テキスト設計用）**：
1. 人 / 働く人 / 仲間（83.1%）
2. 未来 / 挑戦 / 成長（40.8% / 33.8% / 33.8%）
3. 技術（45.1%） / 世界（32.4%）
4. 地域 / 安全 / 安心 / 品質 / 信頼（20-24%）

**避けるべき定型表現**：
- "高品質・低価格" "迅速対応"（コモディティ化）
- "お客様第一" "地域密着" を単独で使う（裏付けなしでは空疎）

### 5-5. 実装スタックの推奨

**モダン（新規構築）**：
- Next.js 14 + TypeScript
- Tailwind CSS + shadcn/ui
- Swiper 11（Hero・実績カルーセル）
- YouTube埋め込み or `<video>` 自動再生
- GSAP 3（必要なら、贅沢装備）

**WordPress（既存資産活用）**：
- Elementor or Breakdance（Gutenberg避ける）
- Swiper plugin
- YouTube embed block

**避けるべき**：
- Lottieの多用（5.6%のみ。実装コスト高）
- Lenis + Swup（贅沢装備）の組合せ（1件のみ）
- Splide（Swiperより後方互換性弱い）
- Webflow（71件中0件）

### 5-6. 生成プロンプト雛形

以下を design-mcp に投入する際の標準プロンプト：

```
業種: <A/B/C/D/E/F/G/Hから選択>
目的: <採用/発注/ブランディング>
会社規模: <大手上場 / 中堅 / 地方中小>
主要サービス: <...>
創業年: <yyyy>
所在地: <都道府県＋市区>
強み1行: <...>
3-4知りナビ: [会社を知る / 仕事を知る / 人を知る / (地域を知る)]
配色: <濃紺系 / ベージュ系 / 赤系 / ブランド色>
英字フォント: Inter
和字フォント: Noto Sans JP
動画: <YouTube URL / なし>
実績数値: <社員数/平均残業/有給取得率/勤続年数>
CTA3段階: [カジュアル面談, 資料DL, エントリー]
業態リネーム案: <元業種を超える新造語>
```

## 7. 結論

### 7-1. R2での最大の発見

**産業LPは"採用LPの別名"**である。71件中74.6%が採用要素を含み、52件が"実績/Works"を持ち、39件が社員インタビューを持つ。「発注元担当者向けの企業紹介」と想定していたR1の前提は、現実には**「求職者向けの自社ブランディング」**であった。

### 7-2. design-mcp向け命題

1. **"10ブロック骨格"は捨てる**。代わりに「**3-4知り構造**」を基軸にする
2. **"濃紺鉄板"も捨てる**。43.7%であって、業種依存。地方ゼネコン=濃紺、大手製薬=ブランド色、BtoC=自由
3. **"人"を前面に**。83%のサイトが「人」を中心に据えている
4. **"業態リネーム"を提案する**。"SMALL GIANTS" "CHEMI-CONNECT" "里山再生企業"のような新造語で既存業種を塗り替える
5. **Next.js + Swiper + Inter + Noto Sans JP** が最安全のモダンスタック
6. **3段階リード漏斗CTA**：カジュアル面談→説明会→エントリー
7. **Hero動画 > Hero写真**。1秒で"動いている会社"を証明
8. **"数字で見る○○"を埋める**。健康/時間/金/成長の順

### 7-3. 未解決課題

R2でも手が届かなかった領域：
- **外部CSS内のメインカラー**: HTML内に記述されない色（外部CSSで管理）は機械抽出できず、一部の代表色は類推に留まる
- **カルーセル内部のコピー**: Swiperの各スライドのコピーは JS 初期化後に見える。現時点ではHTMLから取れない
- **実際のコンバージョン率**: どの設計パターンがCVR高いかは、各社の分析データがないと検証できない
- **A/Bテストの痕跡**: LP同士を比較するだけでは、過去の試行錯誤の結果（現在形のみ）しか見えない

→ R3では**スクリーンショット撮影＋比較**、および**主要10件への実際訪問によるCSS展開済みメインカラー取得**が必要。
